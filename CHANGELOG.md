## [1.5.3] — 2026-07-21

### Додано
- Кастомний спленш-екран при кожному холодному старті застосунку: "Les / Grandes / Sauces" в три центровані рядки шрифтом Great Vibes (білий/помаранчевий, тонкий контур + м'яка тінь, узгоджено з існуючим стилем заголовків у CSS), номер версії шрифтом Play. Реалізовано як HTML-оверлей у `www/index.html` (не нативна картинка) — показується одразу після `window.load`, тримається ~3.5с, потім плавно згасає (1с) і видаляється з DOM.
- Локально підключені шрифти Great Vibes і Play (`@font-face`, `.woff2`) — без залежності від інтернету на пристрої.
- `capacitor.config.json`: додано секцію `SplashScreen` (`launchShowDuration: 0`, `launchAutoHide: false`) — нативний плагін більше не тримає власний екран поверх WebView, натомість ховається одразу викликом `SplashScreen.hide()` на початку скрипта спленша.
- Нативна іконка/фон сплеша згенеровані через `@capacitor/assets` (`npx capacitor-assets generate --android`) — джерело має лежати саме в `assets/splash.png` (не `resources/`, як очікувалось спочатку).

### Відомо
- Android 12+ має власний системний Splash Screen API, який показує лише невелику іконку на кольоровому фоні, ігноруючи будь-яке повноекранне зображення — тому повний дизайн реалізовано на рівні `www/` (HTML/CSS), а не як нативний ресурс; це також дозволяє оновлювати сплеш звичайним OTA, без перезбірки APK.
- Виявлено і враховано: плагін OTA (`@capgo/capacitor-updater`) кешує завантажений бандл у власному сховищі і може "перекривати" свіжий нативний `www/` навіть після `cap sync` — для чистого тестування нативних змін (як-от сплеш) потрібне повне видалення застосунку з пристрою (не просто перевстановлення поверх).

## [1.5.2] — 2026-07-21

### Виправлено
- Кастомна іконка "Білі соуси" (`white-circle`) малювалась контуром (порожнє кільце всередині) — замінено на суцільний білий диск із чорною обвідкою (`icons/icons-data.js`).
- Іконка лупи в головному полі пошуку не відображалась — використовувався клас `ti ti-search` (Tabler Icons), якого немає серед підключених шрифтів (у проєкті лише Font Awesome). Замінено на `fa-solid fa-magnifying-glass`, як і в полі пошуку глосарія.
- Placeholder у полях пошуку зникав лише після вводу символу — додано правило `.sw input:focus::placeholder { color: transparent; }` (`css/styles_a.css`, `css/styles_3.css`), тепер зникає одразу при фокусі на полі.
- Французький термін (`fr`) не відображався в картках глосарія, хоча був у даних — `bGlossary()` (`data/app.js`) не виводив це поле в шаблоні. Додано блок `.gl-fr` і виведення `g.fr`, а також включено `fr` у пошук по глосарію.
- Сторінка "Всі соуси" залишалась порожньою після очищення поля пошуку (хрестиком) — `clearSearch()` передавав порожній рядок `''` напряму в `bAll(filter)`, а `bAll` трактує будь-який визначений аргумент як фільтр категорії (не як пошуковий запит), тому `allFilter` перезаписувався на `''` і список переставав збігатись із будь-якою категорією. Виправлено з двох боків: кнопка очищення більше не передає аргумент (`clearSearch('allq', function(){ bAll(); })`), і сама `bAll()` тепер ігнорує порожній рядок як значення фільтра.

### Відомо
- Розбіжність у кількості соусів (240 проти 242) між локальною версією та GitHub Pages виявилась не багом, а різницею вмісту `localStorage` (ключ `cst`, кастомні соуси) між origin — форма додавання кастомних соусів (`openForm`/`editSauce`) досі є в коді, але прихована з UI (кнопка виклику закоментована); 2 "осиротілі" записи видалено вручну через DevTools на GitHub Pages. Форму вирішено поки не видаляти — можливо знадобиться пізніше, разом із можливим завантажувачем рецептів із .txt.

## v1.3.0 — 2026-07-19

### Added
- New **Glossary** screen (`s-glossary`) — bilingual (EN/UK) culinary terms
  reference with live search across term + all definitions.
  - New data file `data/glossary.js` (`GLOSSARY` array, term/defs structure,
    multiple definitions per term supported).
  - New `bGlossary()` renderer in `data/app.js`, wired into `navTo()`.
  - 5th bottom-nav button (Home / Structure / All / Saved / **Glossary**).
  - New CSS block `.gl-item` / `.gl-term` / `.gl-def`.
  - New offline icon added to `icons/icons-data.js`
    (`streamline:dictionary-language-book-solid`).

### Fixed
- Per-sauce hero photo (`.sauce-hero-img` in `openSauce()`) no longer uses
  CSS `object-fit: cover` (which cropped photos of varying sizes/ratios).
  Now scales via JS (`sizeSauceHero()`, called on `img.onload`):
  - Default: scales to exactly 300px height, width proportional.
  - If that would leave the photo narrower than the screen, scales to
    fill screen width instead (height then exceeds 300px).
  - Centered via `margin: 0 auto` on `.sauce-hero-img`.
